Requisitos: PHP 7.3 o superior Cómo montar este proyecto un servidor local: Por favor clone la carpeta de este repositorio dentro de la carptea htdocs de su servidor local, puede ser xammp --- En su administrador phpmyadmin, por favor cree una base datos con el mismo nombre del proyecto wordpress (En mi caso la carpeta que está dentro del htdocs de mi xammp se llama wordpress y le he puesto el mismo nombre a la base de datos que cree para él). tenga cuenta lo siguiente: usuario: root contraseña: (no tiene, vacía) wp_ --- A la hora de acceder al panel de wordpress localmente en su computador tenga en cuenta que puede acceder mediante los siguientes usuarios: Usuario: envor Contraseña wordpress: p)7uOlv6P1yjeoxqv( Usuario: envor02 Contraseña wordpress: envor2 Usuario: envor03 Contraseña wordpress: envor3 Usuario: envor04 Contraseña wordpress: envor04 CARACTERÍSTICAS: Debe tener un home page que contenga la información principal de la plataforma y muestre los últimos 3 contenidos subidos al blog. El blog debe contener por lo menos 10 contenidos (pueden ser creados desde el plugin Faker Press). Cada entrada del blog debe tener una categoría asociada, para que, en caso de ser necesario, a futuro se pueda crear un template para esa categoría en especial. Cada entrada del blog debe permitir ver metadatos como: Etiquetas Autor Fecha de publicación Categoría de la entrada Se deben tener al menos 4 usuarios creados con roles distintos ya sean administrador, suscriptor, autor, etc. El proyecto debe contener un archivo README en donde se explique cómo puede ser montado en un servidor local para su correspondiente prueba, además, debe estar en un repositorio de GIT público donde pueda ser clonado. ¿Y el diseño? Es un plus pero no será determinante en la evaluación. Nos centraremos en la funcionalidad del proyecto.
